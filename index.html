제안 포인트

1. 검증된 결제 시스템 연동 경험

NICE 본인인증 4개 프로토콜(CheckPlus, SafeKey, NPAC 등) 및 PayGate/Seyfert 결제 연동을 포함한 15건 이상의 외부 API 연동 경험을 보유하고 있습니다. 나이스페이 연동에 필요한 보안 규격과 인증 흐름을 숙지하고 있어 안정적인 구현이 가능합니다.

2. 참고 사이트(chingumobile.com) 분석 기반 제안

현재 운영 중인 사이트를 분석한 결과, XE CMS 기반 PHP 구조에 jQuery + Swiper.js 프론트엔드, ChannelIO 채팅봇이 통합되어 있으며, 4개 언어 다국어 지원이 구현되어 있습니다. 기존 CMS 구조를 최대한 살리면서 결제 모듈과 디자인을 개선하는 방향으로 접근하여 리스크를 최소화합니다.

3. 결제 우선 → 디자인 순차 진행

클라이언트 요구사항에 맞춰 결제 시스템 연동을 우선 완료한 후, 디자인 리뉴얼을 순차적으로 진행합니다. 결제 기능의 안정성을 먼저 확보하여 매출 영향을 최소화합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

본 프로젝트는 외국인 전문 알뜰폰 서비스 웹사이트(chingumobile.com)의 결제 편의성과 사용자 경험을 개선하기 위해 진행됩니다. 현재 사이트는 XE CMS 기반 PHP로 운영 중이며, 4개 언어(한/영/중/베트남어) 다국어를 지원합니다.

핵심 목표:

  • 나이스페이 + PayPal 결제 연동으로 국내외 고객 결제 수단 확장
  • UI/UX 디자인 리뉴얼로 사용자 접근성 및 전환율 향상
  • 기존 시스템 안정성을 유지하면서 점진적 개선

3. 기능 요구사항

3.1 나이스페이 PG 결제 연동

  • [필수] 나이스페이 결제창 연동 (신용카드, 계좌이체, 가상계좌)
  • [필수] 결제 완료 콜백 처리 및 주문 상태 업데이트
  • [필수] 결제 취소/환불 처리 기능
  • [필수] 결제 데이터 MariaDB 저장 및 관리

3.2 PayPal 글로벌 결제 연동

  • [필수] PayPal Checkout 연동 (해외 카드, PayPal 잔액)
  • [필수] 환율 처리 (KRW ↔ USD)
  • [필수] PayPal 결제 완료/취소 콜백 처리
  • [선택] PayPal 정기결제 (구독형 요금제 대비)

3.3 UI/UX 디자인 리뉴얼

  • [필수] 주요 페이지 디자인 개선 (메인, 상품 목록, 상품 상세, 주문)
  • [필수] 반응형 웹 디자인 (PC + 모바일)
  • [필수] 벤치마킹 사이트 참고 디자인 적용
  • [선택] 기존 다국어 UI 일관성 유지

P2P 크라우드 펀딩 플랫폼

핀테크 · 크라우드펀딩 · 2019.12~2020.01

프로젝트 개요

소액 투자자 대상 P2P 렌딩 마켓플레이스의 UI/UX 리뉴얼 및 결제 시스템 운영. 투자금 입출금, 원리금 상환, 정산까지 자동 처리하는 가상계좌 결제 자동화 시스템 구축.

핵심 기능

  • NICE 본인인증 4개 프로토콜(CheckPlus, SafeKey, NPAC) 완전 통합
  • Seyfert/PayGate 결제 연동 — 가상계좌 입출금, 이체, 출금 자동화
  • 15+ 외부 API 연동 (결제, 인증, SMS)
  • 관리자 대시보드 (200+ API 패턴)

본 프로젝트와의 유사점

NICE 계열 PG사 연동 실전 경험이 본 프로젝트의 나이스페이 연동에 직접 활용 가능합니다. 결제 콜백 처리, 환불/취소 로직, 결제 데이터 보안 처리 등 PG 연동의 핵심 패턴을 숙지하고 있습니다. 또한 UI/UX 리뉴얼 프로젝트로서 기존 운영 시스템의 안정성을 유지하면서 개선한 경험이 있습니다.

기술 스택

Python Django MySQL jQuery NICE KYC PayGate
